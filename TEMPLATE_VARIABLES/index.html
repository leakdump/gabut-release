    Enhanced Template Variables for Vulnerability Scanner¶

    This document describes the enhanced template variables system for the vulnerability scanner, which enables dynamic path generation in HTTP requests.

    Overview¶

    The template variables system allows for powerful and flexible vulnerability templates by supporting various types of dynamic variables:

    1. Standard URL Variables: Basic URL components like BaseURL, Host, Path, etc.
    2. Domain Variables: Domain name and subdomain extraction
    3. Time Variables: Current date/time in various formats
    4. Random Variables: Random values for fuzzing and testing
    5. Environment Variables: Access to system environment variables
    6. JavaScript-like Expressions: String manipulation functions

    Available Variables¶

    Standard Variables¶

    Variable Description Example
    {{BaseURL}} The complete URL as specified in the target https://example.com:443/foo/bar.php?id=123
    {{RootURL}} The root URL including scheme, hostname, and port https://example.com:443
    {{Hostname}} The hostname including port example.com:443
    {{Host}} The hostname without port example.com
    {{Port}} The port number 443
    {{Path}} The directory portion of the path /foo
    {{File}} The filename portion of the path bar.php
    {{Scheme}} The URL scheme/protocol https
    {{DN}} Domain name without subdomain example.com
    {{SD}} Subdomain portion www

    Dynamic Variables¶

    Variable Description Example
    {{PathN}} Path segment at index N (zero-based) {{Path0}} → foo, {{Path1}} → bar
    {{PathCount}} Number of path segments 2
    {{Extension}} File extension .php
    {{BaseDomain}} Base domain without subdomains example.com
    {{QueryString}} Query string without ? id=123
    {{HasQuery}} Whether the URL has a query string true or false
    {{TLD}} Top-level domain com
    {{DomainWithoutTLD}} Domain without TLD example
    {{Subdomain}} Full subdomain portion www or api.dev
    {{DomainName}} Same as {{DN}} example.com

    Time Variables¶

    Variable Description Example
    {{DATE}} Current date in ISO format 2025-06-02
    {{TIME}} Current time 15:04:05
    {{DATETIME}} Current date and time 2025-06-02T15:04:05
    {{TIMESTAMP}} Unix timestamp (seconds) 1717027445
    {{UNIXTIME}} Same as TIMESTAMP 1717027445
    {{UNIXTIME_MS}} Unix timestamp in milliseconds 1717027445123
    {{YEAR}} Current year 2025
    {{MONTH}} Current month 06
    {{DAY}} Current day 02
    {{HOUR}} Current hour 15
    {{MINUTE}} Current minute 04
    {{SECOND}} Current second 05
    {{YESTERDAY}} Yesterday's date 2025-06-01
    {{TOMORROW}} Tomorrow's date 2025-06-03
    {{LAST_WEEK}} Date 7 days ago 2025-05-26
    {{NEXT_WEEK}} Date 7 days in the future 2025-06-09
    {{LAST_MONTH}} Date 1 month ago 2025-05-02
    {{NEXT_MONTH}} Date 1 month in the future 2025-07-02
    {{LAST_YEAR}} Date 1 year ago 2024-06-02
    {{NEXT_YEAR}} Date 1 year in the future 2026-06-02

    Random Variables¶

    Variable Description Example
    {{RANDOM_INT}} Random integer between 1000-9999 3752
    {{RANDOM_INT_4}} Random 4-digit integer 0521
    {{RANDOM_INT_8}} Random 8-digit integer 05214783
    {{RANDOM_HEX_8}} Random 8-character hex string a1b2c3d4
    {{RANDOM_HEX_16}} Random 16-character hex string a1b2c3d4e5f6a7b8
    {{RANDOM_HEX_32}} Random 32-character hex string a1b2c3d4...
    {{RANDOM_STRING_8}} Random 8-character alphanumeric string aB3cD7fG
    {{RANDOM_STRING_16}} Random 16-character alphanumeric string aB3cD7fGhJ9kL2mN
    {{RANDOM_STRING_32}} Random 32-character alphanumeric string aB3cD7fG...
    {{RANDOM_LETTERS_8}} Random 8-character letter string aBcDeFgH
    {{RANDOM_NUMBERS_8}} Random 8-character number string 12345678
    {{RANDOM_SPECIAL_8}} Random 8-character special char string !@#$%^&*
    {{RANDOM_MIXED_8}} Random 8-character mixed string a1!b2@c3
    {{RANDOM_UUID}} Random UUID-like string a1b2c3d4-e5f6-a7b8-c9d0-e1f2a3b4c5d6

    Environment Variables¶

    Variable Description Example
    {{ENV_VARIABLE}} Environment variable with ENV_ prefix {{ENV_PATH}}
    {{HOME}} User's home directory /home/username
    {{USER}} Current username username
    {{HOSTNAME}} System hostname server1

    JavaScript-like Expressions¶

    Variables can be manipulated using simple function calls with the syntax: {{variableName.functionName(args)}}.

    Available Functions¶

    Function Description Example Result
    toUpperCase(), upper() Convert to uppercase {{Host.toUpperCase()}} EXAMPLE.COM
    toLowerCase(), lower() Convert to lowercase {{Host.toLowerCase()}} example.com
    trim() Remove whitespace {{Host.trim()}} example.com
    substring(start, end), substr(start, end) Extract substring {{Host.substring(0, 7)}} example
    replace(old, new) Replace text {{Host.replace("example", "test")}} test.com
    split(delimiter, index) Split text {{Host.split(".", 0)}} example
    length(), len() Get string length {{Host.length()}} 11
    reverse() Reverse string {{Host.reverse()}} moc.elpmaxe
    repeat(count) Repeat string {{Host.repeat(2)}} example.comexample.com
    base64encode(), b64enc() Encode as base64 {{Host.base64encode()}} ZXhhbXBsZS5jb20=
    base64decode(), b64dec() Decode from base64 {{Host.base64decode()}} Decoded value
    urlEncode(), urlencode() URL encode {{Host.urlEncode()}} example.com
    hexEncode(), hexencode() Hex encode {{Host.hexEncode()}} 6578616d706c652e636f6d

    Example Templates¶

    Basic Template with Standard Variables¶

    id: basic-example

info:
  name: Basic Template Example
  author: Security Team
  severity: medium

http:
  - method: GET
    path:
      - "{{BaseURL}}/admin"
      - "{{RootURL}}/wp-content/uploads"
      - "{{Scheme}}://{{Host}}:{{Port}}{{Path}}/config.php"

    matchers:
      - type: word
        words:
          - "Admin Panel"
          - "Upload"
        condition: or

    Combined Example with Multiple Variable Types¶

    id: combined-example

info:
  name: Combined Variables Example
  author: Security Team
  severity: medium

http:
  - method: GET
    path:
      - "{{BaseURL}}/{{DATE}}/{{RANDOM_STRING_8}}/{{Host.toUpperCase()}}/{{SD}}"

    headers:
      X-Timestamp: "{{TIMESTAMP}}"
      X-Random: "{{RANDOM_HEX_16}}"
      X-Domain: "{{DN}}"
      X-Path: "{{Path.replace('/', '-')}}"

    matchers:
      - type: status
        status:
          - 200
          - 404
        condition: or

    extractors:
      - type: regex
        part: body
        regex:
          - "token=([a-zA-Z0-9]+)"
          - "user=({{Host}}\\w+)"
        group: 1

    Best Practices¶

    1. Use {{BaseURL}} when you want to append to the exact input URL
    2. Use {{RootURL}} when you want to start from the server root
    3. For maximum flexibility, use individual components like {{Scheme}}, {{Host}}, etc.
    4. Use domain variables ({{DN}} and {{SD}}) to test for domain-specific vulnerabilities
    5. Use time variables to test for time-based issues like backup files and logs
    6. Use random variables for fuzzing and unique payloads
    7. Use expressions to transform values for specific testing scenarios
    8. When defining regex extractors, you can reference template variables in the patterns

    Notes¶

    • All variables are case-sensitive
    • Variables can be used in paths, headers, and body
    • Random values are different for each request
    • Time values are generated at the time of request
    • Expression functions are applied at template processing time, not at request time
